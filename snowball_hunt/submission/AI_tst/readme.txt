* The following concepts have been implement
* A* path searching (for finding path to the different targets and to hiding places)
* Bresenhams Line Drawing Algorithm (for determining if obstacles exist between opponent and character)
*
*  Upon startup of program a map is created so the NPC can navigate its way through the world.
* This map is generated by passing the "ProcessCollision" function and the NPC character
* to the Map Class. The map class then places the character at every available position of the map
* and calls the process collision function. 
* For every position the character is placed  - a MapNode is created. 
* If the Character encounters an obstacle than a state of O is assigned to that node
* If the Character does not collide with obstacles than a state of _ is assigned to that node
* The Map is then passed a list of Gifts that the NPC is protecting. THe GIFTS are designated
* on the map with a G.
* after start up if you look at the DOS command window underneath you will see a map like the following

*
* When not bothered by the human character the NPC does the following
* 1. makes a snowball
* 2. sets out a course for one of the gifts.
* 3. when the NPC reaches the gift it resets its path to the next gift on its list
* 4. this course is repeated until All gifts are visited by the NPC
* 5. NPC restarts is visits to the gifts once it has seen them all.
*
* Mission is Accomplished if all 5 packages are picked up
* Mission is Failed if max damage (100) is taken by the vehicle
* A snowball does 20 damage. (If you are hit 5x mission is failed)
*
* Throwing Range of NPC 10 
* 
* The Path Controller class manages the path of the NPC character
* The NPC considers a range of 20 to be a safe distance from the human character
* the NPC wants to make a snowball while it is a safe distance from the human character
* so you can chase it around the map if it is trying to created a snowball
* this will work until you pin it against a wall/Obstacle. 
* Once pinned it will continue to make its snowball regardless of how close you get.
* 
*	The NPC will always do its best to not run you over ..however if you 
* run over the NPC it will drop its snowball and wait for you to move off of it. 
* 
* The NPC only knows you exist if you are within its throwing radius
* The NPC does not know how close it is to obstacles when it throws its snowball
*	this means that it can throw its snowball only to have it crash into an obstacle
*   just after it releases the snowball.
* 
* The A* path algorithmm is used to find the shortest path to the next GIFT
* and to find a suitable hiding place.
* The A* does not always do the best job - 
* for the two snowmen in the corner the A* algorithm attempts
* to find the shortest path between them only to find it blocked.
* it must then backtrace and go around. 
*
* if you hide by obstacles you will reduce your chances of getting hit.
